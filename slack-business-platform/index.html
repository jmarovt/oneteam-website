Slack platform is a big deal

A lot of current business apps are just glorified HTML forms.

Think about it:

  • Expense reporting software,
  • Bug tracking software,
  • Vacation tracking software,
  • Time tracking software,
  • Employee engagement software,
  • and many more...

As a regular employee, you use these kinds of apps once or twice a month. You get used to play the log in/forgotten password jujitsu. You finally come in, struggle to find the one thing you need to do.

Then finally, at last… you find the form & fill it in. Think about it - this step is the one thing that actually matters. All the stuff before and around is just waste.

Sure, waste that was necessary until now. But waste, nonetheless.

For people on the ground - for people that actually do the work - most non-business critical software is just as a burden. They want to get the work done. They don’t want to learn your new fancy UI. They don’t want to remember yet another login combo.

We are using too much software - And yet too little.

Every office worker has their core productivity software. Software that lives constantly in one of her browser tabs.

  • Photoshop/Sketch… if you’re a designer.
  • CRM… if you’re a sales person.
  • Code editor… if you’re a developer.
  • Hubspot… if you’re a marketer.
  • Text editor… if you’re a copywriter.
  • Accounting software… if you’re in finance.

You get my point. This software, obviously, needs UI.

But if we carefully examine our month, we don’t interact with 3 business apps. If you’re like me, the number is closer to 20.

There’s nothing wrong with using 20 business applications. In fact, I believe we should be using even more software. If it augments us and lets us spend more time doing our core job - I’m all up for it.

Which leads us to...

The rise of invisible software

Invisible software asks me for input, displays some output… and goes away. In some cases, I don’t even need to know that it exists.

For example: I don’t need to love your expense reporting software. I happen to use it because I need to buy some stuff on behalf of my company. So there’s your expense software in the middle that helps me process that.

But I don’t need to love your software. Maybe, I don’t even need to “touch” it.

In fact, expense reporting software will soon run on close to zero touchpoints for most users:

  • It will know when I purchased something. Automatically.
  • It will receive an itemized receipt from the merchant. Automatically.
  • It will know I was in a meeting with a billable client. So it will apply the expense to the client. In case it won’t know, it will ask me to attach a client. Semi-automatically.
  • And then it will push everything to the accounting system. Automatically.

Zero-to-one touchpoint. Invisible software. Coming soon to the business near you.

Why Slack is the invisibility cloak for business software

I believe Slack is pulling us closer to the invisible software reality. Why is that?

At its core, Slack is a simple communication tool: you talk to your colleagues either directly or via shared channels.

But Slack is so much more:

  • Ubiquitous: It’s probably the only software that everyone in the company uses on a daily basis.
  • Used: It’s the one software that people actually check. If you want people to see something - Slack is your best option.
  • Notifications: It has a very reliable cross-device notification system.
  • Developer ecosystem: It has a good set of tools and UI components available to developers. And they’re getting better fast.

Let’s go back to the invisibility theory. Slack provides a platform to run & interact with simple business apps. It has notifications, it has user presence and it has a canvas plus basic UI elements to interact with the user if needed.

This means that a simple business app that only consists of standard HTML layout elements like forms can now run fully in Slack.

It pings you whenever it needs some input. And it presents information upon request or whenever the context makes sense.

The benefits are obvious:

  • Managers: get information from their workers that’s needed
  • Workers: can focus on their core job
  • App developers: can focus on reducing the interaction with users (instead of increasing engagement as its often the case now)

The app (almost) goes away. And this is as close to invisibility as it gets. Welcome to the future!
